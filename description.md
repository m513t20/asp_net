Алгоритм делится на следующие шаги:
1. Сверка позиции (GetSettingsAsync) - грузим настройки и по ним находим необхолимый набор данных используя LoadingSettingsModel и хранящуюся там стартовую позицию.


2. Получаем информацию о журнале, из нее находим новых сотрудников, номенклатуру и категории. Делаем это через нахождение новых ID и их сохранение.

3. Запрашиваем информацию о новых данных. Используется метод `IReader`.`GetRowsById`, который получает данные из клиентской бд.

4. Записываем в БД все полученные данные в PostgreSql используя метод `IWriter`.`PushAsync` или `IWriter`.`Push`.
