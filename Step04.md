# Привет, бро!

Прошлая твоя доработка применена __успешно__ на продуктиве, но к сожалению, нам вновь нужна твоя помощь. Твои умения и экспертиза.
Часть нашей команды считает, что будет верно архитектурно доработать метод **Push**:

```c
    var result = await _loadingService.PushAsync( companyId, transactions, token);
``` 

Расширить данный метод и сделать сразу же обработку и запись данных в бизнес структуры вместо записи в журнал и дальнейшей
пост обработки. У нас в команде идет **серьезный спор** по этому поводу, поэтому, мне необходим эксперт, который выполнит 
на данном этапе доработку. Реализует оба варианта и сделает замеры с тем, чтбы окончательно поставить точку в данном вопросе.

Необходимо:
1. Разработать фоновый процесс который будет выполнять следующее:
   - По транзакциям определять список новых категорий и номенклатуры.
   - Записывать в базу данных новые категории и номенклатуру.
   - По транзакциям определять список новых сотрудников.
   - Записывать в базу данных новых сорудников.
   - Записывать в базу данных новую пачку транзакци с учетом всех связей.

2. Реализовать бизнес логику в виде отдельных сервисов таким образом, чтобы данные
сервисы можно было использовать при рефакторинге  отдельно метода **Push**.

3. Создать набор __модульных тестов__ для бизнес сервисов для опредения выделения новых сущностей.

4. Сделать замер по скорости обработки. Придумать методику измерения скорости и описать ее 
в файле [description.md](./_docs/DESCRIPTION.md)
